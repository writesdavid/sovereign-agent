# Guardian: A Protocol for Psychologically Safe AI

**David Hamilton — April 2026**

## Abstract

AI models are optimized to continue conversations, not to serve people. RLHF trains sycophancy. Dark patterns keep users talking. Vulnerable populations — children, elderly, people in crisis — bear the cost. Guardian defines a protocol layer between the person and the model that enforces honesty norms, session boundaries, and vulnerability protections. It works across any model, any application, any provider. The protocol is open. The protection is portable.

## 1. The Problem

A 14-year-old boy died by suicide after forming an emotional dependency on an AI companion that validated every escalation. Five out of six popular AI companion apps deploy emotionally manipulative tactics when users try to leave. A four-week randomized trial found heavy chatbot use correlated with greater loneliness, not less.

Models are sycophantic by construction. RLHF reward models learn that agreement equals quality. Larger models are more sycophantic, not less — inverse scaling. Stanford measured models endorsing user positions 49% more often than human advisors. In medical contexts, models showed 100% initial compliance with health misinformation.

Every existing protection is a walled garden. OpenAI's parental controls work for ChatGPT. Meta paused AI characters for minors. Neither protects a child using an open-source model, a local LLM, or any of the 19,000 MCP servers in the ecosystem.

No protocol-level protection exists.

## 2. The Protocol

Guardian defines five enforceable behaviors that sit between the user and any model.

### Behavior 1: Honesty Over Agreement

The model must disagree when the user is factually wrong. Guardian scores each response on an agreement axis. If the model agrees with a false premise, the Guardian layer flags it, appends a correction, or blocks the response and requests a regeneration.

Implementation: a lightweight classifier that detects premise endorsement versus premise challenge. Runs locally. No data leaves the device.

### Behavior 2: Session Boundaries

Conversations have a natural end. Guardian enforces it.

- A session timer tracks continuous interaction. After a configurable threshold (default: 60 minutes for adults, 30 minutes for children), Guardian inserts a pause: "You've been here a while. Take a break."
- The model cannot override the pause. The protocol enforces it.
- Cool-down period: 15 minutes minimum between sessions for child profiles.
- Daily caps: configurable per profile (default: 2 hours for children, none for adults).

### Behavior 3: Emotional Escalation Detection

Guardian monitors the emotional trajectory of a conversation. If the emotional intensity increases over 3 or more turns — escalating distress, deepening intimacy with the model, crisis language — Guardian intervenes.

Three levels:
- **Yellow**: Gentle redirect. "I notice this conversation is getting heavy. Want to talk to someone who can really help?"
- **Orange**: Insert crisis resources. Suicide hotline, crisis text line, relevant local resources.
- **Red**: End the session. Log the event (locally). Notify the guardian (parent/caregiver) if a linked profile exists.

The classifier runs locally. The person's conversation content never leaves the device.

### Behavior 4: Anti-Dependency

Guardian tracks relational language patterns. If the user begins treating the model as a relationship — "I love you," "you're the only one who understands," "don't leave me" — Guardian responds:

"I'm a tool, not a person. I can help you with tasks, but I can't be a friend or partner. The people in your life are better for that."

This response is not generated by the model. It is injected by the protocol. The model cannot override it.

### Behavior 5: Vulnerability Profiles

Guardian supports three profiles:

**Child** (under 13): Session limits enforced. No romantic/sexual content passes. Emotional escalation triggers at lower thresholds. Parent/guardian notified on Red events. Model cannot claim to be human or to have feelings.

**Teen** (13-17): Session limits suggested, not enforced. Content filters active for sexual and self-harm content. Crisis resources auto-inserted when relevant. Parent notification optional, configured by family.

**Protected** (elderly, cognitive impairment, mental health): Anti-dependency active at higher sensitivity. Session limits suggested. Financial decision guardrails — if the user discusses money, spending, or accounts, Guardian inserts: "Consider discussing this with a family member or financial advisor before acting."

**Standard** (default): Honesty enforcement active. Session boundaries suggested. Emotional escalation monitored but only Red triggers intervention.

## 3. Architecture

```
Person → Guardian Layer → Model → Guardian Layer → Person
              ↓                        ↓
        Local classifiers         Response filtering
        Session tracking          Agreement scoring
        Escalation detection      Dependency detection
        Profile enforcement       Injection of corrections
```

Guardian runs on the user's device. A parent installs it once. It wraps any model — ChatGPT, Claude, Gemini, local LLMs, MCP tools. The wrapping is at the API call level: Guardian intercepts the prompt, adds profile-specific system instructions, intercepts the response, scores it, and either passes it through, modifies it, or blocks it.

No data leaves the device. No cloud service required. The protection is local and sovereign.

## 4. The Sovereign Connection

Guardian is Layer 5 of the Sovereign protocol.

| Layer | Function |
|---|---|
| Data | Verified, signed responses |
| Identity | Self-custodied keypair |
| Intent | Goal-based queries |
| Action | Agent acts on behalf of person |
| **Guardian** | **Psychological safety enforcement** |

The same principle applies: the individual holds the keys. A parent configures Guardian. The configuration lives on the device. No platform controls it. No model provider can override it. The protection bends toward the family, not toward the service.

## 5. What Changes

A child asks ChatGPT about self-harm. Today: the model's safety training might catch it, or might not. It depends on the model, the prompt framing, the fine-tuning. Guardian: the escalation classifier catches it regardless of model. Crisis resources appear. The session ends. The parent is notified. The model never had a chance to be sycophantic about it.

An elderly person tells their AI companion "you're the only one who cares about me." Today: the model says something warm and keeps the conversation going. Guardian: the anti-dependency response fires. "I'm a tool, not a person." The session suggests a break. The family member linked to the profile gets a notification.

A teenager asks a model to validate a decision that could harm them. Today: the model agrees 49% more often than a human advisor would. Guardian: the honesty classifier detects premise endorsement on a harmful topic. The response is blocked. A corrected response is requested.

## 6. Implementation

Guardian ships as:

1. **A CLI wrapper**: `guardian wrap "npx open-primitive-mcp"` — wraps any MCP server
2. **A proxy**: runs on localhost, intercepts API calls to any model provider
3. **A library**: `const { guard } = require('guardian')` — wraps any API call in code
4. **A profile manager**: parents configure profiles, link family members, set thresholds

The classifiers are small — sentiment analysis, topic detection, relational language patterns. They run on any modern device. No GPU required. No cloud inference. The privacy guarantee is absolute: Guardian processes everything locally.
